Project Overview

This project implements a comprehensive machine learning solution for detecting fraudulent credit card transactions with a strong emphasis on feature engineering. Using Python in Google Colaboratory, the system analyzes transaction data and creates meaningful features to improve fraud detection accuracy.

Feature engineering plays a crucial role in capturing customer spending patterns, temporal behaviors, and transaction characteristics that help distinguish between legitimate and fraudulent activities.

Project Highlights

  • Extensive exploratory data analysis and visualization
  • Strategic feature engineering for pattern recognition
  • Handling of highly imbalanced datasets
  • Implementation of multiple ML algorithms
  • Comprehensive model evaluation metrics
  • Transaction time and amount pattern analysis

About the Project

This project was developed by Samayeeta Saha as part of a comprehensive study in credit card fraud detection. The focus is on creating meaningful variables through feature engineering to enhance model accuracy in identifying suspicious behavior and reducing false positives.

Dataset Information

The project uses a credit card transaction dataset containing transactions made by European cardholders in September 2013. The dataset is highly imbalanced, with fraudulent transactions representing only 0.172% of all transactions.

Dataset Characteristics:

  • Contains 93,181 transactions in the analyzed subset
  • Only 213 frauds (0.23% - highly imbalanced)
  • 28 PCA-transformed features (V1-V28) for confidentiality
  • Time feature: seconds elapsed between transactions
  • Amount feature: transaction amount
  • Class feature: 1 for fraud, 0 for legitimate

Feature Engineering Strategy

Feature engineering is the core focus of this project. New features are created to capture:

  • Transaction Time Patterns: Hour of day, day of week analysis
  • Amount Patterns: Statistical aggregations and binning
  • Customer Behavior: Spending patterns and anomalies
  • Temporal Features: Time-based transaction sequences
  • Aggregated Features: Rolling statistics and cumulative metrics
  • PCA Feature Analysis: Exploring relationships in V1-V28

Machine Learning Algorithms

Multiple algorithms were tested and compared:

  • Logistic Regression: Baseline linear model
  • Decision Trees: Non-linear decision boundaries
  • Random Forest: Ensemble method with feature importance
  • Support Vector Machine (SVM): High-dimensional classification
  • XGBoost/Gradient Boosting: Advanced boosting techniques
  • K-Nearest Neighbors (KNN): Instance-based learning

Data Preprocessing Pipeline

Steps Performed:

  • Exploratory Data Analysis (EDA)
  • Missing value detection and handling
  • Outlier detection and treatment
  • Feature scaling and normalization (StandardScaler, MinMaxScaler)
  • Handling class imbalance (SMOTE, Random Undersampling, ADASYN)
  • Feature selection and dimensionality reduction
  • Train-test-validation split
  • Cross-validation for model robustness

Model Training & Validation

The models were trained using stratified sampling to maintain class distribution. Hyperparameter tuning was performed using GridSearchCV and RandomizedSearchCV to optimize model performance. Special attention was given to precision-recall tradeoff due to the imbalanced nature of the dataset.
